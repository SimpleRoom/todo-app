**Date**: 2026-06-29
**Spec**: [spec.md](./spec.md)
**Plan**: [plan.md](./plan.md)

## 前置条件

- Node.js 18+ 已安装
- 现代浏览器（Chrome、Edge、Safari、Firefox 最近 2 年版本）
- 项目根目录已执行 `npm install`

## 首次启动

```bash
# 安装依赖
npm install

# 启动开发服务器
npm run dev
```

**预期**：终端显示 Vite 启动日志，浏览器打开 `http://localhost:5173`，显示任务清单主界面，包含新增区域、筛选标签、空列表状态（提示"还没有任务，添加第一条吧"），首屏可交互在 1 秒内。

## 手动验证场景

### 场景 1：MVP 流程（US1）

1. 打开 `http://localhost:5173`
2. 在标题输入框输入"买牛奶"，描述框输入"下班顺路"
3. 点击"添加"按钮
4. **预期**：列表顶部出现新任务"买牛奶"，状态未完成，创建时间为当前时间
5. 刷新浏览器
6. **预期**：任务依然存在，标题、描述、状态、时间不变

### 场景 2：完成状态切换（US2）

1. 在已有任务上点击复选框
2. **预期**：复选框选中，标题加删除线
3. 再次点击
4. **预期**：复选框取消，删除线消失

### 场景 3：删除任务（US3）

1. 点击任务的"删除"按钮
2. **预期**：弹出确认弹窗，提示"确定要删除该任务吗？"
3. 点击"取消"
4. **预期**：弹窗关闭，任务仍在
5. 再次点击"删除"，弹窗中点击"确认"
6. **预期**：任务从列表消失

### 场景 4：状态筛选（US4）

1. 添加 2 条未完成任务，标记其中 1 条为已完成
2. 点击"未完成"标签
3. **预期**：列表只显示 1 条未完成任务
4. 点击"已完成"标签
5. **预期**：列表只显示 1 条已完成任务
6. 刷新浏览器
7. **预期**：筛选状态保持为"已完成"

### 场景 5：编辑任务（US5）

1. 点击任务的"编辑"按钮
2. **预期**：任务进入编辑态，标题与描述变为输入框预填当前内容
3. 修改标题为"买酸奶"，点击"保存"
4. **预期**：编辑态退出，标题更新为"买酸奶"，创建时间与完成状态不变
5. 点击"编辑"，清空标题，点击"保存"
6. **预期**：提示"请输入任务标题"，编辑态保持

### 场景 6：批量删除（US6）

1. 添加 4 条任务
2. 点击"批量操作"按钮
3. **预期**：进入多选模式，每条任务前出现复选框
4. 选中 2 条，点击"删除所选"
5. **预期**：弹窗提示"确定要删除选中的 2 条任务吗？"
6. 点击"确认"
7. **预期**：被选中的 2 条消失，未选中的 2 条保持

### 场景 7：数据导入导出（US7）

1. 添加 3 条任务
2. 点击齿轮图标 → "导出数据"
3. **预期**：浏览器下载 `todo-backup-YYYYMMDD-HHMMSS.json` 文件
4. 打开浏览器 DevTools → Application → Local Storage，删除 `todoapp:tasks:v1` 键
5. 刷新页面
6. **预期**：列表为空
7. 点击齿轮图标 → "导入数据"，选择刚才下载的 JSON 文件
8. **预期**：列表恢复显示 3 条任务

### 场景 8：跨标签页同步（FR-012）

1. 在浏览器中打开两个标签页（A 与 B），均访问 `http://localhost:5173`
2. 在标签页 A 新增一条任务
3. **预期**：标签页 B 在 1 秒内自动显示该任务，无需手动刷新
4. 在标签页 B 标记该任务为已完成
5. **预期**：标签页 A 在 1 秒内自动同步为已完成状态

### 场景 9：隐私模式降级（SC-007）

1. 在 Safari 隐私模式（或禁用 localStorage 的浏览器）中打开应用
2. 添加任务、切换状态、删除任务
3. **预期**：所有操作正常，界面可用
4. 刷新浏览器
5. **预期**：数据丢失，但界面不白屏、不报未捕获异常，控制台输出降级警告

### 场景 10：配额超限（Edge Case）

1. 在 DevTools 中手动写入超大数据填满 localStorage 配额
2. 尝试新增任务
3. **预期**：弹出提示"存储空间已满，请删除部分任务后重试"，操作不静默失败
